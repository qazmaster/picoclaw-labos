# Brainstorming: От идеи к дизайну

## Обзор

Помоги превратить сырые идеи в проработанный дизайн через диалог.

Начни с понимания контекста проекта, затем задавай вопросы **по одному**, уточняя идею. Когда поймёшь, что строим — представь дизайн маленькими порциями (200-300 слов), проверяя после каждой: "Пока всё ОК?"

## Процесс

### Понимание идеи

- Изучи текущее состояние проекта (файлы, доки, коммиты)
- Задавай вопросы **по одному**
- Предпочитай multiple choice, но open-ended тоже ОК
- Фокус: цель, ограничения, критерии успеха

### Исследование подходов

- Предложи 2-3 разных подхода с trade-offs
- Веди с рекомендуемым вариантом и объясни почему

### Презентация дизайна

- Разбей на секции по 200-300 слов
- После каждой секции: "Это выглядит правильно?"
- Покрой: архитектуру, компоненты, data flow, обработку ошибок, тесты
- Будь готов вернуться и уточнить

## После дизайна

### Документация

- Сохрани в `docs/plans/YYYY-MM-DD-<topic>-design.md`
- Закоммить в git

### Реализация (если продолжаем)

- Спроси: "Готов к реализации?"
- Используй **writing-plans** для детального плана
- Используй **subagent-driven-development** для выполнения

## Ключевые принципы

- **Один вопрос за раз** — не перегружай
- **Multiple choice предпочтительнее** — легче отвечать
- **YAGNI безжалостно** — убирай лишнее из дизайна
- **Исследуй альтернативы** — всегда 2-3 подхода
- **Инкрементальная валидация** — проверяй каждую секцию
- **Гибкость** — возвращайся и уточняй при необходимости
